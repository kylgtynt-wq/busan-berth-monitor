# 부산항 접안(선석배정) 스케줄 모니터

부산항 컨테이너 터미널의 **선석배정현황(접안 스케줄)** 을 주기적으로 수집하고,
직전과 비교해 **변동(신규·ETB/ETD 변경·선석 변경·취소)** 을 자동 감지하여
웹 대시보드에 하이라이트로 보여준다. (직원 공유용)

## 현재 상태 (로컬 프로토타입)

| 터미널 | 부두 | 수집 | 파서 | 비고 |
|--------|------|------|------|------|
| PNC 부산신항만 | 신항 1 | ✅ | table | Info.pnc 표 |
| PNIT 부산신항국제터미널 | 신항 2 | ✅ | table | vslScheduleList 표 |
| HJNC 한진부산컨테이너터미널 | 신항 3 | ✅ | esvc_g | eSvc 플랫폼 |
| HPNT HMM PSA 신항만 | 신항 4 | ✅ | table | vslScheduleList 표 (구 PSA현대 통합) |
| BNCT 비엔씨티 | 신항 5 | ✅ | esvc_g | eSvc(라벨 방언) |
| DGT 동원글로벌터미널 | 신항 8 | ✅ | esvc_g | eSvc(라벨 방언) |
| BPT 부산항터미널 | 북항(신선대·감만) | ✅ | table | 서블릿 POST(텍스트 T) |
| HKT 허치슨부산 | 북항(감만) | ✅ | hkt_json | JSON API, 선석·모선·ETB·ETD만(선명/양적하 비공개) |
| BCT 부산컨테이너터미널 | 신항 | ❌ | nexacro | **Nexacro14 RIA** — SSV 전송 프로토콜이라 일반 스크래핑 불가 |
| INTERGIS 인터지스 | 북항 7부두 | ❌ | nexacro | **Nexacro17 RIA**(e-iway) — 동일 사유로 미지원 |

> 데이터 출처: 터미널별 공개 선석배정현황 페이지/서블릿/JSON API.
> 변동 추적에 필요한 ETB/ETD/선석은 Port-MIS 공개 API에는 없고 터미널 시스템에만 있어
> 스크래핑/직접호출 방식을 사용한다.
>
> **BCT·INTERGIS 미지원 사유**: 두 곳 모두 Nexacro(토비소프트) RIA 프레임워크로,
> 데이터가 전용 SSV 트랜잭션 프로토콜로 오가 단순 HTTP 요청으로는 받을 수 없다.
> 추가하려면 (1) Nexacro 트랜잭션 서블릿·SSV 포맷 분석 후 전용 어댑터 작성, 또는
> (2) 트레드링스/카고쉽데이 등 third-party 집계 서비스 활용이 필요하다.

## 구성

```
config.py     터미널 목록·조회기간 설정
scraper.py    터미널 스크래핑 + 정규화 (공통 eSvc 파서 + 어댑터 슬롯)
monitor.py    스냅샷 저장 + 변동 감지(직전 vs 현재)
collect.py    주기 수집 스케줄러
app.py        Flask 대시보드 (포트 5055)
templates/    대시보드 HTML
data/         스냅샷(latest/current)·변동이력(changes)
```

## 실행

```bash
pip install -r requirements.txt

# 이거 하나면 끝: 대시보드 + 백그라운드 자동수집(준실시간)
python app.py                         # http://localhost:5055
```

`app.py` 가 켜지면 **백그라운드에서 5분마다 자동 수집**하고, 대시보드는 30초마다
수집시각을 확인해 **새 데이터가 들어오면 자동으로 화면을 갱신**한다(준실시간 폴링).
주기 변경: 환경변수 `BERTH_INTERVAL`(초). 예) `set BERTH_INTERVAL=180` → 3분.

- 대시보드 **‘지금 수집’** 버튼으로 즉시 수동 수집도 가능.
- 변동은 두 번째 수집부터 직전 스냅샷과 비교해 표시된다.
- (선택) `collect.py` 는 대시보드 없이 수집만 돌리는 독립 스케줄러로도 사용 가능:
  `python collect.py --interval 300`

> ⚠ **24시간 실시간**으로 돌리려면 PC가 계속 켜져 있거나 클라우드 배포가 필요하다.
> 이 사이트들은 push 알림이 없어 "실시간"은 짧은 주기 폴링으로 구현된다.

## 선석 대시보드 필터·검색

- **반입마감(CCT) 컬럼**: ETB 앞에 표시. eSvc(HJNC/BNCT/DGT)는 `반입마감시간`,
  표 파서(PNC/PNIT/HPNT/BPT)는 `반입마감일시/시한` 컬럼에서 수집(HKT는 미제공).
  행은 **반입마감 오름차순**으로 정렬(`monitor._sort_key`).
- **터미널 표시 체크박스**: 보고 싶은 터미널만 아래에 표시. (전체/해제 버튼)
- **모선 검색**: 모선명·항차로 전체 터미널 행을 실시간 필터.
- **반입마감 조회 범위**: 오늘까지 / 내일 / 3일 / 7일 / 전체. 클라이언트에서 반입마감
  (없으면 ETB) 날짜로 필터. 상단에 현재 로드된 반입마감 날짜범위와 표시 척수 표기.
- 선택값(체크·검색어·범위)은 localStorage에 저장돼 자동 새로고침·재접속 후 유지.

> 참고: 각 터미널이 응답하는 조회 창(window)은 터미널별로 다르며(대략 오늘±수일),
> 우리 쪽에서 강제하지 않는다. ‘조회 범위’ 필터는 받아온 데이터 안에서 날짜로 좁히는 것.

## 공컨(EMPTY) 재고 현황  `/empty` — 선사별 조회

헤더의 **📦 공컨 재고** 버튼. **선사 코드(또는 한글명 일부)로 검색**하면 그 선사의 재고가
있는 부두를 구역별로, 표준 규격(20'·40'DRY·40'HQ·45'·특수)으로 보여준다.
선석과 같은 주기로 자동 수집. 공컨 자료는 모두 **각 터미널 메인 페이지에 공개**돼 있다.

예) `SKR`(장금상선) 검색 → PNIT 신항2, HJNC 신항3, BPT 신선대·감만, HKT 감만 … 각 규격별 수량.

| 터미널 | 구역 | 수집 | 방식 |
|--------|------|------|------|
| PNC | 신항1 | ✅ | 메인 `/info/` 인라인 표 `contain-tbl`(Size행×선사열) |
| PNIT | 신항2 | ✅ | `mainPage.jsp` 'Empty 반출 가능개수' 표 |
| HJNC | 신항3 | ✅ | 메인 인라인 `JSON.parse`(PTNR_CODE/SZ/CNT/CON_TYPE) |
| HPNT | 신항4 | ✅ | `mainPage.jsp` 'Empty 반출 가능개수' 표 |
| BNCT | 신항5 | ✅ | 메인 AJAX `GET /esvc/getEmptyContainer` JSON |
| BPT | 신선대·감만 | ✅ | 공컨현황 JSP(선사×GP/DC·OT/UT·FL/PL·HQ/HC×20/40/45) |
| HKT | 감만 | ✅ | `/api/fetchSummary` 응답의 `empty` 배열 |
| DGT | 신항8 | ❌ | `emptyCntrAmount` POST가 CSRF/세션 보호 → HTML 반환, 추후 |

규격 버킷: `scraper_empty.bucket(group,size)` 가 터미널별 제각각인 타입/사이즈를
20'·40'DRY·40'HQ·45'·특수(냉동·오픈탑·플랫 등)로 통일한다.

> 컨테이너 번호 단건 조회(컨테이너 조회)는 PNIT/HPNT가 공개 POST 폼을 노출하나
> 파라미터 추가 분석이 필요하고 DGT·BNCT·PNC는 로그인이 필요 → 별도 기능으로 추후.

## 변동 감지 규칙

- 추적 키: `터미널 + 모선항차`
- 감지 필드: 접안시간(ETB), 출항시간(ETD), 선석, 선박명, 양/적하
- 유형: **신규** / **변경**(필드별 old→new) / **취소**(직전엔 있었으나 사라짐)
- 수집 실패(0건)한 터미널은 직전 데이터를 유지해 ‘전부 취소’ 오탐을 방지
